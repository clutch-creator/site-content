# Components

Clutch provides a rich set of components that integrate seamlessly with Next.js and the visual editor. These components are optimized for performance and designed for both visual editing and code-based development.

## Layout Components

### Container

A wrapper component for content with configurable max-width and padding.

```typescript
<Container maxWidth="lg" padding="medium">
  <h1>Content goes here</h1>
</Container>
```

### Grid

Responsive grid layout system:

```typescript
<Grid cols={3} gap="large" responsive>
  <GridItem><Card>1</Card></GridItem>
  <GridItem><Card>2</Card></GridItem>
  <GridItem><Card>3</Card></GridItem>
</Grid>
```

### Flex

Flexible box layout:

```typescript
<Flex direction="row" justify="between" align="center" gap="medium">
  <div>Left</div>
  <div>Right</div>
</Flex>
```

## Media Components

### Image

Optimized image component with automatic optimization:

```typescript
import { Image } from '@clutch-creator/toolkit/components/Image';

// Responsive image with fill
<div className="relative h-96">
  <Image
    src="/hero.jpg"
    alt="Hero image"
    fill
    priority
    className="object-cover"
  />
</div>

// Fixed size image
<Image
  src="/logo.png"
  alt="Logo"
  width={200}
  height={100}
  quality={90}
/>
```

**Props:**
- `src` (string): Image URL or path
- `alt` (string): Alt text for accessibility
- `width` (number): Width in pixels
- `height` (number): Height in pixels
- `fill` (boolean): Fill parent container
- `priority` (boolean): Load image immediately (for above-fold images)
- `quality` (number): Image quality (1-100)
- `className` (string): Additional CSS classes

## Navigation Components

### Link

Enhanced link component with built-in optimization:

```typescript
import { Link } from '@clutch-creator/toolkit/components/Link';

// Internal link
<Link href="/about">About Us</Link>

// External link
<Link href="https://example.com" external>
  Visit Example
</Link>

// With additional params
<Link 
  href="/products" 
  params={{ category: 'books', sort: 'price' }}
>
  View Books
</Link>
```

**Props:**
- `href` (string): Target URL
- `external` (boolean): Open in new tab
- `params` (object): Query parameters
- `className` (string): CSS classes
- `children` (ReactNode): Link content

### Navigation

## Component Configuration

### Registering Components

Register custom components for the visual editor:

```typescript
import { clutchElementConfig } from '@clutch-creator/toolkit';

const CustomButton = ({ children, variant = 'primary' }) => (
  <button className={`btn btn-${variant}`}>
    {children}
  </button>
);

clutchElementConfig(CustomButton, {
  icon: 'üîò',
  styleSelectors: [
    { name: 'Default', value: '&' },
    { name: 'Hover', value: '&:hover' },
    { name: 'Active', value: '&:active' },
    { name: 'Disabled', value: '&:disabled' },
  ],
});

export default CustomButton;
```

### Component Props Types

Use TypeScript for type-safe components:

```typescript
import type { Controls } from '@clutch-creator/toolkit';

type CardProps = {
  title: string;
  description: string;
  image: Controls['Media'];
  /**
   * Button variant
   * @control Select
   */
  variant: 'primary' | 'secondary';
  /**
   * Enable shadow
   * @control Checkbox
   */
  shadow: boolean;
};

export function Card(props: CardProps) {
  // Component implementation
}
```

## Best Practices

### 1. Component Composition

Build complex UIs from simple components:

```typescript
// ‚úÖ Good: Composable
<Card>
  <CardHeader>
    <Heading>Title</Heading>
  </CardHeader>
  <CardBody>
    <Text>Content</Text>
  </CardBody>
</Card>

// ‚ùå Avoid: Monolithic
<Card 
  title="Title" 
  content="Content" 
  showHeader 
  showFooter 
/>
```

### 2. Performance

Use appropriate component types:

```typescript
// Server Component (no 'use client' - default)
async function BlogPost() {
  const post = await fetchPost(); // Runs on server
  return <Article>{post.content}</Article>;
}

// Client Component (needs interactivity)
'use client';
function InteractiveButton() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

### 3. Accessibility

Ensure components are accessible:

```typescript
<Button 
  aria-label="Close modal"
  onClick={handleClose}
>
  <Icon name="close" aria-hidden="true" />
</Button>

<Image 
  src="/hero.jpg" 
  alt="Team working together in office" // Descriptive alt text
/>
```

### 4. Responsive Design

Design mobile-first:

```typescript
<Grid 
  cols={{ mobile: 1, tablet: 2, desktop: 3 }}
  gap={{ mobile: 'small', desktop: 'large' }}
>
  {items.map(item => <Card key={item.id} {...item} />)}
</Grid>
```
